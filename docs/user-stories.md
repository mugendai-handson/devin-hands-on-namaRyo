# ユーザーストーリー v2

## ペルソナ

| ペルソナ | 説明 |
|---------|------|
| ユーザー | アプリケーションの利用者全般。認証、プロフィール設定、テーマ・言語切替を行う |
| エンジニア | タスクを実行する開発者。日常的にカンバンボードを操作する |
| PM（プロジェクトマネージャー） | プロジェクト全体を管理。ダッシュボードで進捗を確認する |
| 管理者 | 組織全体のメンバー管理、監査ログの確認を行う |

---

## 認証（F01）

### US-01: サインアップ
- **As** ユーザー
- **I want to** メールアドレスとパスワードでアカウントを作成したい
- **So that** アプリケーションにログインしてタスク管理を始められる
- **Acceptance Criteria**
  - メールアドレスの形式バリデーションがある
  - パスワードは8文字以上
  - 登録完了後、ダッシュボードにリダイレクトされる
  - 既に登録済みのメールアドレスではエラーが表示される

### US-02: ログイン
- **As** ユーザー
- **I want to** メールアドレスとパスワードでログインしたい
- **So that** 自分のプロジェクトとタスクにアクセスできる
- **Acceptance Criteria**
  - 認証成功でダッシュボードにリダイレクト
  - 認証失敗でエラーメッセージが表示される
  - 未認証でメイン画面にアクセスすると /login にリダイレクトされる

### US-03: パスワードリセット
- **As** ユーザー
- **I want to** パスワードを忘れた場合にリセットしたい
- **So that** アカウントにアクセスできなくなった状態を回復できる
- **Acceptance Criteria**
  - メールアドレスを入力してリセットリンクを送信できる
  - Resend 経由でリセットメールが届く
  - リセットリンクからパスワードを再設定できる
  - リセットリンクには有効期限がある（例: 1時間）

## プロジェクト管理（F02）

### US-04: プロジェクト一覧
- **As** ユーザー
- **I want to** 参加しているプロジェクトを一覧で確認したい
- **So that** 作業対象のプロジェクトを素早く選択できる
- **Acceptance Criteria**
  - 参加中のプロジェクトがカード形式で表示される
  - 各カードにプロジェクト名、タスク数、メンバー数が表示される
  - カードクリックでプロジェクトのカンバンボードへ遷移する

### US-05: プロジェクト作成
- **As** エンジニア
- **I want to** 新しいプロジェクトを作成したい
- **So that** タスクをプロジェクト単位で整理できる
- **Acceptance Criteria**
  - プロジェクト名（必須）と説明を入力して作成できる
  - プロジェクトキーが自動生成される（例: "Devin Task Board" → "DTB"）
  - 作成者が自動的に OWNER になる

### US-06: プロジェクト編集・削除
- **As** プロジェクトオーナー
- **I want to** プロジェクト名や説明を編集、または不要なプロジェクトを削除したい
- **So that** プロジェクト情報を最新に保ち、不要なものを整理できる
- **Acceptance Criteria**
  - プロジェクト設定画面から名前・説明を編集できる
  - OWNER のみプロジェクトを削除できる
  - 削除時に確認ダイアログが表示される
  - 削除するとプロジェクト内の全タスク・コメント等も削除される（CASCADE）

### US-07: メンバー招待
- **As** プロジェクトオーナー
- **I want to** メールアドレスでチームメンバーを招待したい
- **So that** チームでタスクを分担できる
- **Acceptance Criteria**
  - メールアドレスを入力してメンバーを追加できる
  - 権限（ADMIN / MEMBER / VIEWER）を設定できる
  - 招待されたメンバーのプロジェクト一覧に表示される

### US-08: メンバー権限管理
- **As** プロジェクト OWNER / ADMIN
- **I want to** メンバーの権限を変更したり、メンバーを削除したい
- **So that** プロジェクトへのアクセスを適切に管理できる
- **Acceptance Criteria**
  - メンバー一覧から権限を変更できる（OWNER / ADMIN のみ操作可能）
  - メンバーをプロジェクトから削除できる
  - OWNER は自分自身を削除できない
  - VIEWER はタスクの閲覧のみで編集できない

## タスク管理（F03）

### US-09: タスク作成
- **As** エンジニア
- **I want to** タスクをタイトル・説明・優先度・期限付きで作成したい
- **So that** やるべき作業を明確に記録できる
- **Acceptance Criteria**
  - タイトル（必須）、説明（Markdown）、優先度、担当者、期限、カテゴリを設定できる
  - タスク番号が自動採番される（DTB-1, DTB-2...）
  - 作成後にタスク詳細画面が表示される
  - 起票者（reporter）が自動的に記録される

### US-10: タスク詳細の確認・編集
- **As** エンジニア
- **I want to** タスクの詳細を確認し、各フィールドをインラインで編集したい
- **So that** 画面遷移なしで素早くタスク情報を更新できる
- **Acceptance Criteria**
  - タイトル、説明、ステータス、優先度、担当者、期限をインライン編集できる
  - 説明は Markdown プレビューが表示される
  - 変更がリアルタイムで保存される
  - 変更時にアクティビティログが記録される

### US-11: タスク削除
- **As** エンジニア
- **I want to** 不要なタスクを削除したい
- **So that** プロジェクトのタスク一覧を整理できる
- **Acceptance Criteria**
  - 確認ダイアログが表示される
  - 削除すると関連するコメント・添付ファイル・サブタスクも削除される（CASCADE）
  - VIEWER 権限のユーザーは削除できない

### US-12: サブタスク
- **As** エンジニア
- **I want to** タスクにサブタスクを追加したい
- **So that** 大きなタスクを細かく分解して進捗を管理できる
- **Acceptance Criteria**
  - 親タスクの詳細画面からサブタスクを作成できる
  - サブタスクのステータスが一覧表示される
  - サブタスクにも担当者・期限を設定できる

## カンバンボード（F04）

### US-13: カンバン表示
- **As** エンジニア
- **I want to** タスクをカンバンボード形式で一覧したい
- **So that** 各ステータスのタスク状況を一目で把握できる
- **Acceptance Criteria**
  - BACKLOG / TODO / IN_PROGRESS / IN_REVIEW / DONE の5カラム表示
  - 各カードにタイトル、番号、優先度バッジ、担当者アバター、期限が表示される
  - カラムごとにタスク数が表示される

### US-14: ドラッグ＆ドロップ
- **As** エンジニア
- **I want to** タスクカードをドラッグ＆ドロップでステータス変更したい
- **So that** マウス操作だけで素早くタスクの進捗を更新できる
- **Acceptance Criteria**
  - カラム間の D&D でステータスが更新される
  - カラム内の D&D で並び順が更新される
  - ドロップ先がハイライトされる
  - 楽観的更新で即座にUIに反映される
  - ステータス変更時にアクティビティログが記録される

### US-15: カンバンのクイック作成
- **As** エンジニア
- **I want to** カンバンのカラム上から素早くタスクを作成したい
- **So that** 画面遷移なしで思いついたタスクをすぐに追加できる
- **Acceptance Criteria**
  - 各カラムに「+」ボタンが表示される
  - タイトルを入力するだけでタスクが作成される
  - 作成されたタスクはそのカラムのステータスで作成される
  - 作成後、タスクカードがカラム内に即座に表示される

## タスク一覧（F05）

### US-16: リストビュー
- **As** PM
- **I want to** タスクをテーブル形式で一覧表示したい
- **So that** 複数のタスクを比較・ソートして全体像を把握できる
- **Acceptance Criteria**
  - タスク番号、タイトル、ステータス、優先度、担当者、期限、カテゴリが表示される
  - カラムヘッダークリックでソートできる（昇順/降順）
  - 行クリックでタスク詳細画面へ遷移する
  - ページネーション（20件/ページ）

---

## ガントチャート（F06）

### US-17: ガントチャート表示
- **As** PM
- **I want to** タスクのスケジュールをガントチャートで表示したい
- **So that** 期間の重なりや全体のスケジュール感を視覚的に確認できる
- **Acceptance Criteria**
  - 開始日〜期限を横棒で表示
  - 日/週/月の表示切替
  - 今日の日付を示すバーティカルライン
  - タスクの横棒をドラッグで期限変更
  - カテゴリ別、担当者別のグルーピング表示

---

## ダッシュボード（F07）

### US-18: ダッシュボード
- **As** PM
- **I want to** プロジェクト横断のサマリーを一画面で確認したい
- **So that** 全プロジェクトの健全性を把握し、ボトルネックを特定できる
- **Acceptance Criteria**
  - サマリーカード: 総タスク数、完了数、進行中数、期限超過数
  - ステータス別棒グラフ、優先度別円グラフ（recharts）
  - 直近7日間の完了タスク数の推移グラフ
  - 自分にアサインされたタスク上位5件
  - 期限が近いタスク上位5件
  - 最近のアクティビティフィード上位10件

---

## コメント（F08）

### US-19: コメント投稿
- **As** エンジニア
- **I want to** タスクにコメントを投稿したい
- **So that** タスクに関する議論や情報共有をタスク内で完結できる
- **Acceptance Criteria**
  - Markdown でコメントを投稿できる
  - @メンションで特定ユーザーに通知を送れる
  - 自分のコメントを編集・削除できる
  - コメント投稿時にアクティビティログが記録される

---

## カテゴリ（F09）

### US-20: カテゴリ管理
- **As** PM
- **I want to** タスクにカテゴリ（バグ、機能追加、改善等）を付与したい
- **So that** タスクを種類別に分類・フィルタリングできる
- **Acceptance Criteria**
  - プロジェクトごとにカテゴリを作成（名前 + カラー）
  - カテゴリの編集・削除ができる
  - タスクに複数カテゴリを付与できる
  - カテゴリでフィルタリングできる
  - デフォルトカテゴリ: バグ（赤）、機能追加（青）、改善（緑）、ドキュメント（紫）

---

## ファイル添付（F10）

### US-21: ファイル添付
- **As** エンジニア
- **I want to** タスクにファイルを添付したい
- **So that** スクリーンショットや資料をタスクに紐付けて共有できる
- **Acceptance Criteria**
  - 画像（jpg, png, gif, webp）/ PDF / テキスト / Office をアップロードできる（上限10MB）
  - 画像はサムネイルプレビュー表示
  - ダウンロード・削除ができる
  - アップロード時にアクティビティログが記録される

---

## 検索（F11）

### US-22: グローバル検索
- **As** エンジニア
- **I want to** タスクをキーワードで横断検索したい
- **So that** 目的のタスクを素早く見つけられる
- **Acceptance Criteria**
  - ヘッダーの検索バーから検索できる
  - タイトル、説明、番号、コメント内容が検索対象
  - インクリメンタルサーチ（デバウンス300ms）
  - 検索結果のハイライト表示
  - ステータス、優先度、担当者、カテゴリ、期限範囲でフィルタリングできる（AND条件）

---

## アクティビティログ（F12）

### US-23: タスクの変更履歴
- **As** エンジニア
- **I want to** タスクの変更履歴を確認したい
- **So that** 誰がいつ何を変更したかを追跡できる
- **Acceptance Criteria**
  - タスク詳細画面の「アクティビティ」タブに時系列で表示
  - フィールド単位の変更前後が表示される（例: ステータス: TODO → IN_PROGRESS）
  - 記録対象: 作成、更新、ステータス変更、担当者変更、コメント投稿、ファイル添付

### US-24: プロジェクトのアクティビティフィード
- **As** PM
- **I want to** プロジェクト全体のアクティビティを一覧で確認したい
- **So that** チーム全体の作業状況をリアルタイムで把握できる
- **Acceptance Criteria**
  - プロジェクト単位で全タスクのアクティビティを時系列表示
  - ユーザーアバターとアクション内容が表示される
  - ダッシュボードのアクティビティフィードにも上位10件が表示される

---

## 通知（F13）

### US-25: アプリ内通知
- **As** エンジニア
- **I want to** 自分に関連するアクションの通知を受け取りたい
- **So that** タスクのアサインやコメントを見逃さない
- **Acceptance Criteria**
  - ヘッダーのベルアイコンに未読数バッジ
  - トリガー: タスクアサイン、コメント、ステータス変更、期限1日前、@メンション
  - 通知一覧画面で既読/未読切替、一括既読
  - 通知クリックで該当タスクへ遷移

### US-26: メール通知
- **As** エンジニア
- **I want to** 重要な通知をメールでも受け取りたい
- **So that** アプリを開いていない時でも把握できる
- **Acceptance Criteria**
  - アプリ内通知と同じトリガーで Resend 経由でメール送信
  - ユーザーごとにメール通知の ON/OFF 設定可能
  - メール内に該当タスクへのリンクを含む

## 多言語対応（F14）

### US-27: 言語切替
- **As** ユーザー
- **I want to** UIの言語を日本語/英語で切り替えたい
- **So that** 自分が使いやすい言語で操作できる
- **Acceptance Criteria**
  - プロフィール設定で言語を切り替えられる
  - 全UI要素（ボタン、ラベル、メッセージ、エラー文）が翻訳される
  - 日付フォーマットがロケールに対応する（ja: YYYY年MM月DD日 / en: MMM DD, YYYY）
  - 選択がDB保存され永続化される

---

## ダークモード（F15）

### US-28: テーマ切替
- **As** ユーザー
- **I want to** ライト/ダーク/システム設定のテーマを選びたい
- **So that** 自分の好みや環境に合った表示で使える
- **Acceptance Criteria**
  - ヘッダーまたはプロフィール設定から切替可能
  - 選択がDB保存され永続化される
  - 全画面・全コンポーネントがテーマに対応する

---

## 監査ログ（F16）

### US-29: 監査ログ閲覧
- **As** 管理者
- **I want to** 全ユーザーの操作履歴を閲覧したい
- **So that** セキュリティ上の問題や不正操作を追跡できる
- **Acceptance Criteria**
  - 管理者（ADMIN ロール）のみアクセス可能
  - 記録対象: ログイン/ログアウト、プロジェクト作成/削除、メンバー追加/削除、権限変更
  - ユーザー、操作種別、日付範囲でフィルタリング
  - IP アドレス、ブラウザ情報を記録
  - テーブル形式でページネーション付き表示

---

## 複数ユーザー・プロフィール（F17）

### US-30: プロフィール設定
- **As** ユーザー
- **I want to** 自分の名前、アバター、言語、テーマを設定したい
- **So that** 自分のアカウントをカスタマイズできる
- **Acceptance Criteria**
  - 名前、アバターURL、言語（ja/en）、テーマ（ライト/ダーク/システム）を変更できる
  - 変更が即座に反映される
  - 設定がDB保存され永続化される
